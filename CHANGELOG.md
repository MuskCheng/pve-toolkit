## [V0.5.20] - 2026-02-23

### 优化

- **简化容器创建流程**
  - 移除创建容器后询问是否预装 Docker 的提示
  - 用户可通过 Docker 管理或安装常用工具功能按需安装

- **安装常用工具提示**
  - 添加提示信息："请先创建 LXC 容器，或使用已有容器"

## [V0.5.19] - 2026-02-23

### 新增

- **安装常用工具菜单**
  - LXC 容器管理菜单新增"安装常用工具"选项
  - 新增 Lucky V2 一键安装功能
  - 预留更多工具扩展位置

- **Lucky V2 安装功能**
  - 支持自定义端口、配置目录、容器名称
  - 默认值回车确认，简化操作
  - 自动检测 Docker 环境，未安装时提示安装
  - 优先从离线包加载镜像，失败时回退在线拉取
  - 安装完成后显示访问地址

## [V0.5.18] - 2026-02-23

### 新增

- **Docker 离线安装功能**
  - Docker 管理菜单新增"安装 Docker (离线包)"选项
  - 支持从 GitHub Releases 下载离线包（含 Docker、Docker Compose、Lucky V2）
  - 支持 GitHub 代理加速下载（国内推荐）
  - 支持本地离线包安装

- **自动离线包构建工作流**
  - 添加 GitHub Actions 工作流自动检测 Docker/Lucky 更新
  - 有更新时自动构建并发布到 Releases
  - Release 说明中包含各组件版本信息

### 变更

- **系统要求调整**
  - 最低 PVE 版本从 9.0 调整为 9.1
  - 最低 Debian 版本保持为 13 (Trixie)

## [V0.5.17] - 2026-02-23

### 新增

- **创建容器时选择容器类型**
  - 支持选择特权容器或无特权容器
  - 显示两种容器类型的功能差异说明
  - 默认选择特权容器（Docker 兼容性最佳）

- **容器类型转换功能**
  - 容器操作菜单新增"转换容器类型"选项
  - 支持特权容器 ↔ 无特权容器互转
  - 特权→无特权转换时显示警告并自动转换文件所有权
  - 转换过程自动停止/启动容器

- **GitHub Actions 自动发布**
  - 添加 CI 工作流，VERSION 文件更新时自动创建 Release

## [V0.5.16] - 2026-02-22

### 新增

- **网络配置详细显示**

  - DHCP 模式下自动获取容器实际 IP 地址
  - 容器未运行时显示提示信息

  - 使用分割线替代表格框线，避免终端显示错位
  - 简化输出格式，提高兼容性

## [V0.5.15] - 2026-02-22

### 新增

- **Docker 容器管理功能**
  - Docker 管理菜单新增"容器管理"选项

  - 添加 Docker Compose 部署结果验证

  - 定义 WHITE 颜色变量

## [V0.5.14] - 2026-02-22

### 新增

- **Docker 换源功能**
  - Docker 管理菜单新增"换源"选项
  - 支持多个国内镜像源：阿里云、中科大、网易、腾讯云、Docker中国
  - 支持自定义镜像源
  - 自动配置 /etc/docker/daemon.json
  - 自动重启 Docker 服务并验证

### 修复

- **Docker Compose 路径问题**
  - 使用完整路径调用 docker-compose（/usr/local/bin/docker-compose）
  - 解决容器内 PATH 未包含 /usr/local/bin 导致命令找不到的问题

## [V0.5.13] - 2026-02-22

### 修复

- **Docker Compose 安装稳定性修复**
  - 使用固定版本 v2.24.0 替代动态获取最新版本（避免 v5.x 文件名不兼容）
  - 增加下载文件完整性验证：
    - 检查文件大小 > 1MB
    - 检查文件头是否为 HTML（避免 404 页面）
    - 使用 `file` 命令验证 ELF 可执行文件格式
    - 验证 `--version` 输出包含 "Docker Compose"
  - 验证失败时自动删除无效文件

## [V0.5.12] - 2026-02-22

  - 添加文件路径检测作为备用（`/usr/bin/docker`, `/usr/local/bin/docker-compose`）
  - 解决 LXC 容器中 PATH 未正确设置导致命令检测失败的问题

- **Docker Compose 安装改进**
  - 添加 curl/wget 安装状态变量检测
  - 显示 curl/wget 安装结果
  - 使用绝对路径验证 docker-compose 安装

## [V0.5.11] - 2026-02-22
